# Finance Operations

## 1) 報價標準流程
1. 需求訪談（功能、角色、整合、風險）
2. 複雜度分級（L1-L4）
3. 工時估算（開發/設計/維運/PM）
4. 風險係數與利潤目標
5. 產出正式報價與里程碑

## 2) 定價包設計
- 建站包：基礎展示站、商業站、平台站
- 維運包：安全更新、備份、監控、故障處理
- 行銷包：SEO、內容、廣告追蹤、報表
- 客製包：WordPress 外掛、流程系統、第三方整合

## 3) 收款追蹤與逾期提醒
- 預設付款節點：50/30/20（啟動/里程碑/上線驗收）
- D+3 提醒：友善通知
- D+7 提醒：暫停新需求排程
- D+14 提醒：暫停高風險部署
- 所有提醒須有紀錄（日期、內容、回覆）

## 4) 每客戶毛利計算（實際流程）
1. 收入：合約金額 + 加購 + 維運月費
2. 直接成本：外包、人力工時、第三方 API、雲資源
3. 間接成本：管理、工具、行政分攤
4. 毛利 = 收入 - 直接成本 - 間接分攤
5. 每月檢視：低毛利案調價、縮範圍、或調整交付模式

## 5) 多幣別與跨國收款
- 每份 SOW 指定合約幣別
- 報價有效期需標註（建議 7-14 天）
- 匯率波動超過門檻時觸發重估機制
- 稅務/VAT 規則由合約條款明確定義

## 6) 財務例行節奏
- 每週：收款狀態檢查
- 每月：客戶毛利表 + 逾期表
- 每季：定價策略調整
