# Outsourcing Playbook

## 1) 角色分工
- AM：客戶窗口與需求確認
- PM：排程與資源協調
- Tech Lead：技術設計與品質把關
- Executor：開發/設計/行銷/自動化執行

## 2) 任務交接流程
1. 任務建立（背景、目標、範圍、期限）
2. 指派與估時
3. 中繼檢查（風險與阻塞）
4. 交付（成果 + 測試證據 + 回滾說明）
5. 驗收與結案

## 3) 驗收標準（DoD）
- 功能符合需求
- 測試通過（至少含 happy path + edge case）
- 文件已更新（操作、部署、限制）
- 可回滾方案可執行

## 4) 外包評分維度
- 準時率
- 品質與返工率
- 溝通效率
- 風險意識

## 5) 評分制度
- 評分規則與加權請依 `docs/operations/outsourcing-vendor-scorecard.md`
- 每個里程碑結束後都需評分並記錄

## 6) 接手最小文件
- 任務背景與目標
- 目前進度
- 未完成項
- 風險與依賴
- 下一步動作
