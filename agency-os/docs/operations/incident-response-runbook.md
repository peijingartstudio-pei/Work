# Incident Response Runbook

## 1) 事件分級
- P1：資料外洩、網站不可用、重大安全事件
- P2：核心功能部分失效
- P3：一般功能異常或效能下降

## 2) 標準流程
1. 偵測與建單
2. 分級（P1/P2/P3）
3. 通報（內部與客戶）
4. 隔離（停用風險功能、封鎖來源、切流）
5. 修復與驗證
6. 服務恢復
7. 復盤與改進

## 3) 時效目標
- P1：15 分鐘內啟動事件處理
- P2：60 分鐘內完成初步處置
- P3：下一個維護時段處理

## 4) 證據保存
- 事件時間線
- 相關日志與快照
- 影響範圍
- 變更與操作紀錄

## 5) 事件後輸出
- Incident Report
- Root Cause
- 修復措施與預防措施
- SOP 更新項目
