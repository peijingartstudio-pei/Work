# Leads and Scraping Compliance Checklist

## Objective
- Ensure lead generation and web data collection activities meet legal and platform requirements.

## Legal Baseline
- [ ] Verify lawful basis for data processing
- [ ] Confirm permitted use of collected data
- [ ] Confirm retention and deletion policy
- [ ] Confirm region-specific obligations (GDPR/CCPA/local law)

## Source and Platform Rules
- [ ] Terms of service reviewed for each source
- [ ] Robots and rate limits respected
- [ ] No bypass of technical protections
- [ ] No scraping of restricted personal data

## Operational Controls
- [ ] Data minimization applied
- [ ] Access control enforced
- [ ] Processing logs retained
- [ ] Incident response path documented

## Client Contract Controls
- [ ] Scope and data sources explicitly approved in SOW
- [ ] CR required for new source or new data class
- [ ] Liability and compliance ownership clarified

## Audit Evidence
- [ ] Source register
- [ ] Data flow map
- [ ] Consent/notice records (if required)
- [ ] Deletion evidence
