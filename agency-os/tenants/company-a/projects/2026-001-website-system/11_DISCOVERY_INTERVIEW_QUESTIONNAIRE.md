# Discovery 訪談問卷 - 2026-001

## 使用方式
- 目標：一次收齊進入 Build 前必備資訊
- 會議長度建議：45~60 分鐘
- 記錄規則：未確認事項一律標記「待確認」

## A. 決策與溝通機制（必答）
1. 專案最終決策者是誰？（姓名/職稱/聯絡方式）
2. 日常執行窗口是誰？備援窗口是誰？
3. 需求確認與簽核流程為何？（誰提、誰審、誰核）
4. 例行同步頻率與固定會議時段？
5. 緊急事件通報群組與優先聯絡順序？

## B. 商業與品牌定位（必答）
1. 目前品牌的核心價值與差異點是什麼？
2. 本站前三大目標受眾是誰？
3. 受眾最常見的需求/痛點是什麼？
4. 本案最重要的商業目標（最多 3 項）？
5. 成功指標排序（曝光、詢問、預約、成交、留存）？

## C. 範圍確認（必答）
1. 本期一定要上線的頁面與功能有哪些？
2. 本期明確不做哪些項目？
3. 是否包含會員、付款、預約、課程等交易型流程？
4. 內容上稿由誰負責？初版素材何時交付？
5. 多語系需求是否為本期範圍？

## D. 工具與權限（必答）
1. WordPress 主機/網域/DNS 權限由誰持有？
2. GitHub repo 建立與成員權限分配方式？
3. Supabase 專案是否新建？誰保管管理權限？
4. n8n 是否使用共用環境或客戶專屬環境？
5. DataForSEO / Replicate 本期是否啟用？若不啟用，何時評估？

## E. 變更與估價規則（必答）
1. CR 估價偏好：套裝價、工時價、或混合？
2. CR 簽核需要哪些資訊？（工時/成本/風險/時程）
3. 可接受的變更處理 SLA？
4. 哪些情境可列為緊急插單？
5. 是否有月度預留工時或維運點數制度？

## F. 里程碑與時程（必答）
1. 目前 M1/M2/M3/M4 日期是否可接受？
2. 客戶端素材與回饋時限（工作天）？
3. 是否有不可動檔期（活動/節慶/投放節點）？
4. UAT 參與人員與驗收責任分配？
5. Go-live 後 7 天觀測窗口可配合時段？

## G. 風險與依賴（必答）
1. 目前最可能延遲專案的三個因素？
2. 需第三方配合的事項與窗口？
3. 法規/合規/隱私要求是否有額外限制？
4. 是否有既有系統資料需要搬遷？
5. 目前尚未決定但會影響架構的議題？

- [ ] 品牌定位與受眾已確認

- [ ] 權限交付時程已確認
- [ ] CR 估價基準已確認
